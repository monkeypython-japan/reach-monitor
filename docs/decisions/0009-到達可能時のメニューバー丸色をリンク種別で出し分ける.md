# ADR-0009: 到達可能時のメニューバー丸色をリンク種別で出し分ける

- ステータス: 承認 (Accepted)
- 日付: 2026-07-11

## コンテキスト

[[0006-メニューバーアイコンにnsimage-istemplate-falseを使用|ADR-0006]] により、メニューバーの
丸は到達可能=赤・到達不可=黒・確認中=グレーで色分けされていた。今回、
[[0008-nwpathmonitorで使用中のリンク種別を判定する|ADR-0008]] で「使用中のリンク種別
（Wi-Fi/Ethernet）」を取得できるようになったことを受け、メニューバー上でも
Wi-Fi 接続時は青丸、Ethernet 接続時は赤丸と、一目でリンク種別が分かる表示にしたい
という要望が出た。

ただし単純に丸の色をリンク種別だけに置き換えると、従来 ADR-0006 が担っていた
「到達可能かどうかが色で一目で分かる」という機能が失われてしまう。

## 検討した選択肢

- **採用案**: 到達可能 (`reachable`) の場合のみ、丸の色をリンク種別で出し分ける
  （Wi-Fi = 青、Ethernet 等 = 赤）。確認中はグレー、到達不可は黒のまま変更しない。
  つまり「グレー/黒＝状態異常、青/赤＝到達可能かつリンク種別」という二段構えにする。
- **却下案**: 丸の色を常にリンク種別のみで表示し、到達性はポップオーバーでのみ確認する
  — 却下理由: メニューバーだけで到達性が分からなくなり、本アプリの主目的
  （キャプティブポータル等での到達性の一目確認）を損なう。

## 決定

`ReachMonitorApp.menuBarIcon` と `AppState.menuBarCircleColor` を次のロジックに変更する。

```swift
switch status {
case .checking:    color = 灰
case .unreachable: color = 黒
case .reachable:   color = isLinkWiFi ? 青 : 赤
}
```

`AppState.isLinkWiFi`（`link.interfaceType == .wifi`）を新設し、リンク種別判定を
一箇所に集約する。

## 結果

### 利点

- 到達不可/確認中は従来通り一目で分かる。
- 到達可能時は、Wi-Fi か Ethernet かも同時にメニューバーだけで分かるようになる。

### 欠点・トレードオフ

- 色の意味が状態によって変わる（灰/黒＝状態、青/赤＝リンク種別）二段構えになり、
  ADR-0006 時点よりも色の意味がやや複雑になる。
- Wi-Fi/Ethernet 以外のリンク種別（セルラー等）で到達可能な場合も赤色になる
  （Ethernet と区別しない）。
