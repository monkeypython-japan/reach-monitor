# 位置情報の許可が再署名でリセットされることがある

- ステータス: 対応しない (wontfix)
- 日付: 2026-07-01

## 症状

`bundle/make-app.sh` を再実行して `.app` を再ビルド・再署名した後、以前は許可されていたはずの
位置情報アクセスが再度ダイアログで求められることがある。SSID/BSSID が一時的に取得できなく
なったように見える。

## 原因

このアプリは ad-hoc 署名 (`codesign --sign -`) を使っており、ad-hoc の signing identity は
実行ファイルの内容（ハッシュ）から導出される。TCC（位置情報・通知などの権限管理）は
署名 identity に紐付いて許可を記録しているため、コードを変更して再ビルド・再署名すると
identity が変わり、macOS からは「別のアプリ」に見えて許可がリセットされる。

## 回避策 / 対処

恒久的な対処はしない（wontfix）。Apple Developer 証明書のような安定した identity で
署名すれば発生しないが、[[0001-swiftpmビルドと手動appパッケージングでad-hoc署名を採用|ADR-0001]]
により個人利用の範囲でそのコストは見合わないと判断済み。再ビルド後に位置情報プロンプトが
再度出るのは想定内の挙動として受け入れ、許可し直せばよい。

**再評価条件**: 正式な Developer ID 証明書での署名（[[0001-swiftpmビルドと手動appパッケージングでad-hoc署名を採用|ADR-0001]]
の却下案 B）を採用することになった場合は、この issue は解消される見込みなのでクローズしてよい。
